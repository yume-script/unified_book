# 📚 통합 도서 검색 (V1.1.5)

BookOasis 미디어 서버에서 코어 수정 없이 메타데이터 검색과 라이브러리 관리를 안정적으로 지원하는 통합 도서 검색 플러그인 패키지입니다. 알라딘, 네이버, 구글의 도서 데이터를 스레드 풀(Thread Pool) 기반으로 병렬 조회하여 높은 신뢰도의 도서 정보를 고속으로 완성합니다.

---

## ✨ 주요 특징 (Key Features)

| 기능 | 설명 |
| :--- | :--- |
| **⚡ 초고속 병렬 검색** | 알라딘, 네이버, 구글 도서 API를 동시에 백그라운드로 호출하는 `ThreadPoolExecutor` 기술을 가동하여 전체 검색 속도를 비약적으로 단축시킵니다. |
| **🖼️ 고해상도 커버 수집** | 원본에 가까운 표지 품질 확보를 위해 소스별 이미지 주소를 정제하여 가장 선명한 최고 화질의 커버를 다운로드합니다. |
| **🏷️ 정돈된 UI 레이아웃** | 소개글 본문(Description) 영역에 서지 정보를 중복 노출하여 지저분해지는 현상을 방지하고, 시스템이 출력하는 `출간일` 영역 끝자리에 `[ISBN: ...]` 값을 결합하여 깔끔한 한 줄 라인으로 출력합니다. |
| **🤖 하이브리드 LLM 연동** | 구글 공식 **Gemini 3.5 Flash-Lite** 모델을 네이티브로 활용하거나, 사용자가 자체 구동하는 **LiteLLM 프록시 게이트웨이(OpenAI 호환)** 및 마스터 키(Bearer 토큰)를 설정창에서 손쉽게 연동하여 유연하게 판권지를 분석합니다. |
| **🎯 정밀 매칭 원천 추적** | ISBN 정밀 검색 성공 시 어떤 경로로 도서를 식별했는지 타이틀 접두사에 명확히 구분해주며, 끝자리에는 별표(`*`)를 기입하여 식별성을 강화합니다.<br>- `[알라딘/ISBN]` : 직접 ISBN을 입력해 검색함<br>- `[네이버/DB]` : 로컬 데이터베이스의 ISBN 기록을 불러옴<br>- `[출처/LOCAL]` : 책 파일의 메타데이터/XHTML을 직접 파싱함<br>- `[출처/AI]` : 로컬 분석 실패 후 제미나이/LiteLLM AI가 본문을 판독함 |
| **🔐 기존 ISBN 데이터 보호** | 신규 검색 결과에 ISBN 정보가 비어 있는 경우에도, 기존 데이터베이스에 저장되어 있던 소중한 ISBN 데이터가 공백으로 덮어씌워져 지워지지 않도록 `COALESCE / NULLIF` 안전 쿼리를 수행합니다. |
| **📂 임포트 섀도잉 방지** | 다른 플러그인이나 코어에 `utils.py`가 존재하여 이름이 충돌하는 문제를 방지하기 위해 파일명을 `utils_unified.py`로 변경하고, **절대 경로 기준 동적 모듈 로드(`importlib`)** 방식을 도입하여 작동의 안전성을 보장합니다. |
| **⚙️ DB Row 크래시 원천 차단** | 데이터베이스 게이트웨이가 반환하는 Row 객체에서 `.get()`을 직접 호출할 때 발생할 수 있는 `AttributeError` 예외를 감지하여, 안전하게 데이터 검색 및 저장 구문을 실행하는 내부 헬퍼(`get_row_val`)를 내장했습니다. |

## 💡 문서 내부 ISBN 정밀 역추적 기술

*   **PDF 스캔 대역 및 인코딩 보완:**
    *   최근 단행본들의 목차나 머리말이 길어지는 트렌드에 발맞추어 스캔 대역을 **앞쪽 30페이지 및 뒤쪽 30페이지**로 대폭 확장하여 깊숙이 숨어있는 판권지(Colophon)를 안정적으로 찾아냅니다.
    *   PDF 내부의 특수 유니코드 대시 기호(en-dash `–`, em-dash `—`, soft-hyphen `\u00ad` 등)와 구분 기호(`.`)를 분석 직전 표준 하이픈(`-`)으로 실시간 강제 치환하는 전처리를 적용하여 정규식 검출 누락을 방지합니다.
*   **EPUB XHTML 엔티티 복원:**
    *   EPUB 파일 조판 시 공백 줄바꿈 깨짐을 방지하기 위해 사용되는 웹 특수 공백 부호(`&nbsp;`, `&#160;` 등)를 온전한 공백 문자로 언이스케이프(Unescape) 처리하는 HTML 파서를 내장했습니다.
    *   판권지가 책 머리말(표지 직후) 영역에 있는 경우를 대비하여 **앞쪽 8개 책장 및 뒤쪽 8개 책장**을 동시에 이중 교차 분석합니다.
*   **유연한 부분 일치 검색:**
    *   데이터베이스 내 도서명과 파일명이 미세하게 불일치하더라도 검색어의 앞 '두 단어'를 파싱하여 부분 일치 매칭(`LIKE %단어%`)하는 3단계 백업 쿼리를 구동, 실제 파일 추적률을 유지합니다.
*   **2단계 자동 백업 검색 (Fallback):**
    *   ISBN 검색 결과가 0건이거나 실패한 경우 즉시 전처리 정제된 원래 책 제목 검색으로 Fallback 전환되어 신뢰할 수 있는 도서 검색 결과를 보장합니다.

## 📂 디렉토리 구조 (Directory Structure)

GitHub 배포를 위한 폴더 구조 구성은 다음과 같습니다. 해당 플러그인은 아래 8개의 파일로 긴밀하게 결합되어 구동됩니다.

```text
plugins/metadata/unified_book/
├── __init__.py           # 플러그인 로드 진입점
├── unified_book.py       # 메인 코어 플러그인 (흐름 제어 및 DB 트랜잭션)
├── aladin.py             # 알라딘 검색 (일반 검색 및 ISBN 전용 API)
├── naver.py              # 네이버 검색 (일반 검색 및 d_isbn 전용 API)
├── google.py             # 구글 검색 (일반 검색 및 Google Volume API)
├── utils_unified.py      # 공통 유틸리티 (날짜 정제, 고화질 수집, 문서 파싱, DB 헬퍼, LLM)
├── index.html            # 가로형 맞춤 설정 화면 마크업
├── style.css             # 어두운 가로형 맞춤 설정 화면 스타일
└── VERSION               # 자동 업데이트 추적용 버전 파일

---

## 📦 설치 및 적용 가이드 (Installation)

1.  `plugins/metadata/` 경로 하부에 `unified_book` 폴더를 생성하고 패키지 파일들을 배치합니다.
2.  BookOasis 웹 UI의 환경설정 > 플러그인 메뉴에서 `Unified BOOK Search` 활성화 스위치를 켭니다.
3.  알라딘 TTBKey 및 네이버 Client API 값을 설정하고 저장을 누릅니다.
4.  책 상세 정보 수정 > 돋보기 아이콘(검색) 클릭 시 알라딘/네이버/구글의 정돈된 검색 결과를 즉시 조회하실 수 있습니다.
